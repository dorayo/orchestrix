# Decision Files Optimization Summary

## 优化目标
控制决策数据文件在 100 行左右，减少 LLM token 消耗，同时保持精度不丢失。

## 优化策略

### 1. 合并相似规则
- 使用条件表达式 (IN, AND, OR) 合并逻辑相似的规则
- 减少重复的 metadata 字段
- 示例：将 13 条规则合并为 4-5 条核心规则

### 2. 精简 Output 定义
- 移除冗长的字段描述
- 只保留类型定义和关键属性
- 使用简洁的内联语法 `{type: string}`

### 3. 压缩 Examples
- 从 7-8 个示例减少到 3-4 个关键场景
- 只保留输入输出的核心字段
- 移除冗余的 reasoning 重复

### 4. 简化 Usage Context
- 从详细的多段说明压缩为 2-3 句核心要点
- 移除重复的协调说明
- 保留关键的状态转换和 handoff 逻辑

### 5. 缩短 Reasoning
- 保留核心决策逻辑
- 移除重复的解释性文字
- 每条 reasoning 控制在 1-2 句话

## 精度保证

所有优化都确保：
- ✅ 决策逻辑完整保留
- ✅ 关键条件和结果不变
- ✅ 必要的 metadata 保留
- ✅ 核心示例覆盖主要场景
- ✅ 使用说明简洁但完整

## Token 节省估算

假设平均每个 token 约 4 个字符：
- 优化前：2985 行 × 平均 80 字符/行 ≈ 60,000 tokens
- 优化后：1135 行 × 平均 80 字符/行 ≈ 23,000 tokens
- **节省约 37,000 tokens (62%)**

## 维护建议

1. **新增决策文件**：参考优化后的格式，保持简洁
2. **规则数量**：尽量控制在 5-8 条核心规则
3. **示例数量**：3-4 个关键场景即可
4. **描述长度**：每个字段描述控制在 1 行
5. **定期审查**：每季度检查是否有新的冗余可以精简
