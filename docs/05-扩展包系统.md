# Orchestrix 扩展包系统

## 什么是扩展包？

扩展包是Orchestrix框架的专业化模块，通过添加特定领域的AI代理来扩展核心功能。每个扩展包都包含专业的代理角色、任务模板和知识库，让Orchestrix适用于软件开发以外的各种领域。

### 核心优势
- 🎯 **保持核心精简** - 核心框架专注软件开发，扩展包处理其他领域
- 🧠 **专业化深度** - 每个扩展包提供特定领域的深度专业知识
- 🔧 **按需安装** - 只安装需要的扩展包，避免系统臃肿
- 🌐 **社区驱动** - 任何人都可以创建和分享扩展包

## 使用扩展包

### 安装扩展包
```bash
# 查看可用扩展包
npx orchestrix list-packs

# 安装特定扩展包
npx orchestrix install-pack game-development

# 验证安装
npx orchestrix status
```

### 在Web界面使用
```bash
# 加载扩展包配置
# 上传：dist/expansion-packs/[pack-name]/team-config.yaml

# 使用扩展包代理
*game-designer    # 切换到游戏设计师
*level-designer   # 切换到关卡设计师
```

### 在IDE中使用
```bash
# 调用扩展包代理
@game-designer.mdc    # Cursor中调用游戏设计师
/mobile-ux           # Claude Code中调用移动UX设计师
```

## 创建自定义扩展包

### 第一步：定义领域
- 确定专业领域和解决的问题
- 分析目标用户和使用场景
- 定义扩展包的边界和范围

### 第二步：设计代理角色
每个代理应包含：
```yaml
# agents/example-agent.md
name: "专家名称"
role: "具体职责"
expertise: "专业技能"
personality: "沟通风格"
knowledge_base: "知识范围"
```

### 第三步：创建任务模板
```yaml
# tasks/example-task.md
name: "任务名称"
description: "任务描述"
steps: "执行步骤"
inputs: "需要输入"
outputs: "预期输出"
```

### 第四步：配置扩展包
```yaml
# config.yaml
name: "扩展包名称"
version: "1.0.0"
description: "功能描述"
agents: ["代理列表"]
tasks: ["任务列表"]
dependencies: ["依赖包"]
```

### 第五步：测试和优化
- 使用真实场景测试
- 收集用户反馈
- 完善代理响应
- 优化任务流程
