# 🤖 Part 2: Orchestrix 怎么解决 (10分钟)

## 核心理念（3个类比）

| 理念        | 游戏语言                     |
| ----------- | ---------------------------- |

| 标准 > 随意 | 副本攻略 ≠ 瞎打              |

## 10个专业 Agent

### 规划层（类似策划组）

| Agent     | 等于     | 干什么             |
| --------- | -------- | ------------------ |
| Analyst   | 市场调研 | 竞品分析、用户画像 |
| PM        | 主策划   | 写PRD、定功能      |
| UX-Expert | UI设计   | 出交互稿           |
| Architect | 主程序   | 技术方案           |
| PO        | 制作人   | 质量把关           |

### 执行层（类似开发组）

| Agent | 等于     | 干什么        |
| ----- | -------- | ------------- |
| SM    | 项目管理 | 拆任务        |
| Dev   | 程序员   | 写代码        |
| QA    | 测试     | 审代码、找Bug |

## 智能决策矩阵（现场画）

```
质量高 + 复杂度低 → 直接开发 ✅
质量高 + 复杂度高 → 技术审核 🔍
质量低           → 打回重写 ❌
```
